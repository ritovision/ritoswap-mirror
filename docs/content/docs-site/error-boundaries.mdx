# Error Boundaries

The docs site wraps pages in a custom error boundary so runtime failures render a branded recovery UI instead of a generic crash.

## Usage pattern

The boundary uses <code>fallbackRender</code> to show <code>PageError</code> and logs the error to the console for debugging.

<Callout type="info">
  If you need an inline error, use <code>ErrorShellInline</code> in the component itself instead of throwing to the boundary.
</Callout>
